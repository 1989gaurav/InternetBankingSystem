Use Case: Draft Request



                                                Fig- 4.1.4 Draft Request Use case.


Following are the specifications for this Use Case-

Actors:
·     Customers, Entrepreneurs, Industrialists, Organizations, Administrators, etc.

Pre-Conditions
·     User is already logged in.

Post-Conditions
·     Draft is created and the draft number is provided to the user.

Basic Flow
·     The user will enter the necessary details.
·     The user will be asked for his Transaction password.
·     On confirmation, the draft request will be sent to the server.
·     The server will return the draft number to the user.

Alternate Flow : Insufficient Account balance
·     The User will be informed that the account has insufficient balance
·     Control will be returned to the first page.
