Here is the Testing Plan for IBS:

·     First we did Static Testing for each and every module where we checked that the Code was written according to the Conventions adopted by us. For this, we used the Eclipse Plug-in already installed in Eclipse. At this stage we also started making comments for creating the Java Docs in the format specified by the conventions.

·     Next Testing for a module was the Black Box Testing where we checked for all valid & invalid inputs for every module and verified if we got the desired result(s) or not.

·     Next is the White Box Testing where we glanced through the code once & then checked for some suitable inputs where the program is expected to crash or generate Exceptions and report bugs if any.

·     Next was the Unit Testing wherein we used JUnit.

·     Requirements Testing is the next phase wherein we checked whether logically the modules satisfied its requirements or not. In this stage we might need to add more fields etc.

·     The last two testing stages was Integration Testing & Regression Testing. These were done when all the above said testing were done for all the modules. In Integration testing we tested whether the modules depending on each other worked properly or not. In Regression Testing we re-executed all the tests performed earlier to check if the Modules worked correctly or not.
